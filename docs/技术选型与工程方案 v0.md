# 技术选型与工程方案 v0

更新时间：2026-06-11

## 一句话结论

首版建议采用 `Next.js + TypeScript + Tailwind CSS + shadcn/ui + Motion + lucide-react`，工程代码放在 `web/` 子目录，根目录保留 `docs/`、`images/`、`Video/` 等策划与原始资产。这样可以避免在已有资料目录中强行初始化 Next.js，也方便后续 Vercel 把 Root Directory 指向 `web`。

## 官方依据

| 技术 | 官方资料 | 本项目采用点 |
|---|---|---|
| Next.js | https://nextjs.org/docs/app/getting-started/installation | 使用 `create-next-app`、TypeScript、Tailwind、ESLint、App Router、`src/`、`@/*` alias |
| shadcn/ui | https://ui.shadcn.com/docs/installation/next | 在 Next.js 项目中运行 `shadcn init` 并按需添加组件 |
| Motion for React | https://motion.dev/docs/react-installation | 使用 `motion` 包，从 `motion/react` 导入 |
| lucide-react | https://v0.lucide.dev/guide/packages/lucide-react | 使用树摇友好的 React SVG 图标 |

## 工程目录决策

推荐结构：

```text
TJ_Official_Website/
  docs/
  images/
  Video/
  web/
    src/
    public/
    package.json
```

为什么不直接在根目录初始化：

- 当前根目录已经承担“策划文档 + 原始资产库”的角色；
- `docs/`、`images/`、`Video/` 里有大量非工程资产；
- `create-next-app .` 在非空目录中更容易遇到冲突；
- `web/` 子目录更清晰，部署时配置 Root Directory 即可。

## 包管理器

推荐使用 `pnpm`。

原因：

- Next.js 和 shadcn 官方文档都提供 pnpm 命令；
- 安装速度快；
- lockfile 清晰；
- 后续 workspace 扩展更方便。

如果本机没有 pnpm，先启用：

```bash
corepack enable
corepack prepare pnpm@latest --activate
```

验证：

```bash
node -v
pnpm -v
```

Next.js 官方当前要求 Node.js 至少 `20.9`。

## 初始化命令

在仓库根目录执行：

```bash
pnpm create next-app@latest web --yes
```

官方默认会启用：

- TypeScript；
- Tailwind CSS；
- ESLint；
- App Router；
- Turbopack dev；
- `@/*` import alias；
- 推荐的 Next.js 默认配置。

如果需要显式选择 `src/` 目录，可使用交互式初始化：

```bash
pnpm create next-app@latest web --src-dir
```

交互选择建议：

| 提示 | 选择 |
|---|---|
| TypeScript | Yes |
| Linter | ESLint |
| React Compiler | No，首版保守 |
| Tailwind CSS | Yes |
| `src/` directory | Yes |
| App Router | Yes |
| import alias | `@/*` |
| AGENTS.md | Yes，如果 CLI 提示 |

## shadcn/ui 初始化

进入工程目录：

```bash
cd web
pnpm dlx shadcn@latest init
```

初始化选择建议：

| 项 | 建议 |
|---|---|
| style | New York 或默认 |
| base color | Neutral |
| CSS variables | Yes |
| components path | `@/components` |
| utils path | `@/lib/utils` |

首批添加组件：

```bash
pnpm dlx shadcn@latest add button sheet dialog aspect-ratio separator badge tooltip
```

如果首版确定做轻量意向表单，再添加：

```bash
pnpm dlx shadcn@latest add input textarea form label
```

可延后组件：

```bash
pnpm dlx shadcn@latest add accordion skeleton
```

暂不添加：

- `pagination`
- `data-table`
- `calendar`
- `command`
- `carousel`

## 额外依赖

安装 Motion 和图标：

```bash
pnpm add motion lucide-react
```

使用方式：

```ts
import { motion, useReducedMotion } from "motion/react"
import { Play, Menu, ArrowRight } from "lucide-react"
```

说明：

- Motion 是 Framer Motion 后续推荐包名；
- 本项目只用轻量入场、reveal、hover、Dialog/Sheet 过渡；
- lucide-react 只按需导入图标，避免动态导入全部图标。

## 资源放置规则

原始资产保留在根目录：

```text

```

工程使用资产复制到：

```text
web/public/
  brand/
  media/
  images/
  social/
```

首版需要复制：

```bash
mkdir -p web/public/media web/public/images/sketches
cp 'Video/天机优选 - 01.mp4' 'web/public/media/brand-film-original.mp4'
cp 'images/Style reference/视频封面.png' 'web/public/media/brand-film-poster.png'
cp 'images/草图/homepage-pc-full-direction.png' 'web/public/images/sketches/homepage-pc-full-direction.png'
```

注意：

- `brand-film-original.mp4` 只用于本地开发验证；
- 上线前应生成压缩版，例如 `brand-film.mp4`；
- 不要在代码里引用带中文或空格的源路径；
- 不覆盖原始文件；
- Logo、社媒、联系方式、备案先留空。

## 申请会员页策略

首版采用轻量意向提交，不跳商城。

开发策略：

- `/apply` 先做真实页面；
- 表单字段极少；
- 没有隐私说明和处理方式前，不真实收集信息；
- 可先做 `ApplyFormPlaceholder`；
- 后续改商城跳转时，只替换 `ApplyIntentForm` 或按钮 action。

建议字段，待确认：

| 字段 | 是否建议 |
|---|---|
| 姓名/称呼 | 建议 |
| 手机号/微信 | 二选一，需隐私说明 |
| 所在城市 | 可选 |
| 了解方式/备注 | 可选 |

## Git 与版本管理建议

当前 `.gitignore` 忽略了：

```text
images/
docs/
video/
```

我的建议：

1. `docs/` 应该纳入版本管理。  
   这些文档已经是项目决策资产，后续开发、复盘、交接都要依赖。

2. `images/` 和 `Video/` 可以继续不纳入 git。  
   原始图片和视频体积大、变动多，更适合本地资产库或后续单独放对象存储。

3. 工程里的 `web/public/` 是否纳入 git 要分层判断。  
   小图、封面、图标可以纳入；154MB 原始视频不建议纳入。

推荐调整 `.gitignore`：

```gitignore
.meta-team
.meta-team-runs/
.playwright-mcp
.spec-workflow
.env
images/
Video/
video/
web/public/media/brand-film-original.mp4
```

也就是说：移除 `docs/` 忽略，让文档进入 git；保留原始图片/视频忽略。

如果暂时不想改 `.gitignore`，也可以把文档复制到一个被跟踪的目录，例如 `project-docs/`。但更简单的是直接允许 `docs/` 被跟踪。

## 首次开发执行顺序

1. 决定是否调整 `.gitignore`，让 `docs/` 进入版本管理。
2. 初始化 `web/` Next.js 工程。
3. 初始化 shadcn/ui。
4. 安装 `motion` 和 `lucide-react`。
5. 复制指定素材到 `web/public/`。
6. 落 tokens 和全局 CSS。
7. 实现 `SiteHeader`、`HomeHero`、`BrandFilmSection`、`HomeEntrances`、`QuietClosing`。
8. 启动浏览器检查首屏和视频模块。
9. 实现剩余首页区块。
10. 实现 `/apply` 轻量意向页。
11. 做桌面和移动端截图复核。
12. `pnpm lint`、`pnpm build`。

## 开发前 Stop Rule

以下条件未满足时，不应声称“官网首版完成”：

- `/` 和 `/apply` 都能访问；
- Header 和移动菜单可用；
- Brand Film 使用指定封面，视频点击播放；
- `/apply` 不误导用户已经注册成功；
- Logo、社媒、联系方式、备案缺失时不展示假内容；
- build 通过；
- 桌面和移动端截图检查通过。
