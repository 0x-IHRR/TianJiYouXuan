# 内容资产清单 v0

更新时间：2026-06-11

## 一句话结论

当前项目已有“品牌影片 + 暖色风格参考 + 八张页面草图 + 指定视频封面”，足够支撑首页第一版视觉方向，但还不足以支撑一个完整高信任官网。首版开发必须明确保留 Logo、正式文案、社媒链接、联系方式、真实线下空间与供应链证据等缺口，不能用 mock 数据填满。

## 资产使用原则

| 原则 | 说明 |
|---|---|
| 真实优先 | 真实视频、真实空间、真实产品、真实服务证据优先于图库和 AI 假图 |
| 缺什么留什么 | 缺 Logo、社媒、联系方式、申请入口时，文档和代码中标记待补，不编造 |
| 草图只做方向 | AI 草图只用于布局和氛围参考，里面的文字、Logo、品牌符号不可直接使用 |
| 参考图不等于最终图 | Style reference 用于色调、光感、材质和情绪，不直接当官网内容图 |
| 视频要优化 | 当前视频体积较大，开发时应考虑封面、懒加载、压缩和播放体验 |
| 不做假活动墙 | 没有真实活动/空间/人物素材时，不展示“会员活动照片墙” |

### 使用建议

- 不作为首屏背景视频；
- 放在第二屏 `BrandFilmSection`；
- 默认展示静态封面图，点击后再播放；
- 视频不自动播放声音；
- 移动端避免自动加载完整视频；
- 开发前建议从视频中截取 1-3 张封面候选图；
- 如部署到公网，建议压缩出 web 友好版本，例如 1080p 或更低码率版本。

## 资产命名建议

后续进入工程后，建议整理到 `public/`：

```text
public/
  brand/
    logo.svg
    favicon.ico
  media/
    brand-film.mp4
    brand-film-poster.jpg
  images/
    hero/
    lifestyle/
    trust/
    sketches/
  social/
    wechat.svg
    xiaohongshu.svg
    douyin.svg
    bilibili.svg
```

命名规则：

- 使用英文小写和连字符；
- 不在代码中引用带空格的素材文件名；
- 原始素材保留在 `images/`、`Video/`；
- 工程使用的压缩素材进入 `public/`；
- 不覆盖原始文件。
